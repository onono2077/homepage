# とにかく仕組み化をしていきたい

今年の目標は「積み重ね」と「挑戦」。<br>
最近得た情報として、習慣化には仕組みを作ることが重要というものがあった。<br>

勉強にしろ生活習慣にしろ、着手するハードルをとにかく下げることで継続につながるということだ。

確かに。<br>

私は完全主義な節があるため、なんでもより高いレベルを目指してしまう。<br>

悪いことではないが、少しの妥協もしたくないあまりに着手するハードルはとにかく高かった。<br>
3日坊主はおろか、目標や計画を立てた次の日には辞めてしまうことが多い。

日々の小さな継続を目標としている今年は、この「仕組み」化を実践していきたい。

まず、このブログ。

自作ホームページの記事の執筆になかなか筆が進まなかったが、<br>
最近始めた github での日記をとりあえずそのまま載せればいいのではと思いついた。

日記は github に毎日草を生やしたいと思っていたところ、「TIL」という文化を知って書き始めた。

TILとは「Today I Learned」の略で<br>
その日に学んだことを短く・気軽に・継続的に記録する文化のことだそうだ。<br>
GitHub に TIL リポジトリを作って学びを積み上げていく。

エンジニアとして実力をつけたいと思っているにも関わらず<br>
成長を自覚できないのはアウトプット不足も大きいと思い、学んだことに限らず書き連ねていくことにした。

その日の日付のmdファイルを作成し、日付が変わる前に日記リポジトリにpushする。

一方で自作ホームページのリポジトリは別にある。

AIに2つの異なるリポジトリのファイルを同期させることはできるか聞いたところ<br>
githubのワークフロー機能を使えば可能とのこと。

ワークフロー機能は初めて使ったが、AIに聞きながら設定・mdファイルをホームページのリポジトリに反映することができた。

すごく面白い..！

こんなことができるのか！という驚きと、ホームページに自動反映されるという便利さによる感動があった。<br>
この業界に足を踏み入れた時と変わらない、好奇心をくすぐる刺激がやっぱりある。

「仕組みを作る」ということに対して、自動化＝楽に・雑に、のようなイメージがあったが<br>
逆に、着手のハードルを下げることで目標や課題に向き合う回数が増え、取り組む行為そのものに集中しやすくなることに気付いた。

この「面白い」「次はこんな風にできないかな」「他にどんなことがができるのだろう」という気持ちを保ったまま<br>
知識の吸収やアウトプットにつなげるには<br>

「仕組み化」いいかも。

なんなら「仕組み化」そのものも楽しい。<br>
生活にも取り入れていきたい。
